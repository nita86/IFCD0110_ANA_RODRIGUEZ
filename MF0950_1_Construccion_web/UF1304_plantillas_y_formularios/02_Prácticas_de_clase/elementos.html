El elemento <input>

El elemento de formulario más importante es el <input>elemento. El <input>elemento se puede mostrar en varias formas, dependiendo del type atributo. Ejemplo: <input name="firstname" type="text"> Si el "type" se omite el atributo, el campo de entrada obtiene el tipo por defecto: "texto".

HTML5 <datalist> Element

El <datalist>elemento especifica una lista de opciones predefinidas para un <input>elemento. Los usuarios podrán ver una lista desplegable de las opciones predefinidas, ya que los datos de entrada. El listatributo del <input>elemento, debe hacer referencia al idatributo del <datalist>elemento.

CODIGO

<form name="datalist" action="/action_page.php" method="get" accept-charset="UTF-8" autocomplete="on" enctype="multipart/form-data" target="_blank">
  <input list="browsers">
  <datalist id="browsers">
    <option value="Internet Explorer">
    <option value="Firefox">
    <option value="Chrome">
    <option value="Opera">
    <option value="Safari">
  </datalist> 
</form>

HTML <label> Tag.

La etiqueta <label> define una etiqueta para un botón <>, <input>, <metro>, <salida>, <progreso>, <seleccionar> o elemento <textarea>.

El elemento <label> no hace como algo especial para el usuario. Sin embargo, proporciona una mejora la facilidad de uso para los usuarios de ratón, ya que si el usuario hace clic en el texto dentro del elemento <label>, se alterna el control.

El atributo de la etiqueta <label> debería ser igual al atributo id del elemento relacionado para unirse juntos.

CODIGO

<form action="/action_page.php">
  <label for="male">Male</label>
  <input type="radio" name="gender" id="male" value="male"><br>
  <label for="female">Female</label>
  <input type="radio" name="gender" id="female" value="female"><br>
  <label for="other">Other</label>
  <input type="radio" name="gender" id="other" value="other"><br><br>
  <input type="submit" value="Submit">
</form>
